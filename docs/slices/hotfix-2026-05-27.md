# Hotfix 2026-05-27 — 修复两个线上问题

## 角色
DeepSeek TUI (外部修复)

## 做了什么

### 1. profile_card「Cannot read properties of undefined (reading 'length')」crash

**根因**: `AIEngine.buildCards()` 对 WR 类型（`candidate_card` / `profile_card` / `analytics`）的兜底字段只有 9 个，LLM 返回 `profile_card` 时如果未包含 `skills` 字段，`data.skills` 为 `undefined` → `ProfileCard` 将其传给 `<TagGroup>` → `TagGroup` 访问 `tags.length` → crash。

**修复**:
- `AIEngine.buildCards()`: WR 兜底从 9 个字段扩展到 19 个，覆盖 `skills` / `careerHistory` / `notes` / `name` / `currentCompany` / `currentTitle` 等
- 新增处理 LLM 不按规范返回 `data` 包裹的情况：自动从卡片平级字段构造 `c.data`
- `TagGroup.tsx`: `safe = tags ?? []` 防御

### 2. 刷新页面后历史消息重放打字机效果

**根因**: `MessageBubble` 对所有 agent 消息都从 `''` 开始逐字打字，未区分新消息和历史消息。刷新后从 localStorage 加载的历史消息也全部重新打一遍。

**修复**: 引入 `isFresh = Date.now() - message.timestamp < 3000` 判定。超过 3 秒的历史消息直接显示完整内容，仅真正的新消息走打字机动画。
