## 1. 四态规范理解

contracts/cards.ts 定义了 `CardMode = 'loading' | 'empty' | 'error' | 'demo' | 'live'`，5 种 mode。我的映射方案：

| contracts mode | 视觉表达 | 文案 |
|---|---|---|
| `loading` | 骨架屏：灰色圆角块 + shimmer 微动画，占位与对应卡片结构一致 | 无文案 |
| `empty` | 灰色插画（空状态插图）+ `empty_hint` 文本 + 可选引导 action | contracts.empty_hint（LLM 生成） |
| `error` | `border-red-400/30 bg-red-50/5` 浅红边框 + 错误图标（`AlertTriangle`）+ `error_hint` + "重试"按钮 | contracts.error_hint（LLM 生成） |
| `live` | 完整渲染真实数据（对应 S3 文档的 `ready`） | payload 内容 |
| `demo` | 渲染演示数据，**右上角显示"演示数据"小标签**（dev mode 可全局显隐） | 预设 mock 数据 |

**关于 demo 标识的澄清**：S3 文档描述的是 `isDemo` boolean 叠加在任意 mode 上显示标识。但 contracts 把 `demo` 作为独立 mode。我倾向于二者并存：`mode='demo'` 时强制渲染演示数据 + 显示标识；`mode='live'` 时 `is_demo=true` 也可叠加标识（比如在 dev 环境下演示真实数据流）。请确认。

## 4. CardRenderer 路由实现

采用 **静态 map + switch 的 O(1) 路由**：

```tsx
// src/components/cards/CardRenderer.tsx
import type { AgentCard } from '@/contracts/cards';
import { C1_CandidateList } from './C1_CandidateList';
// ... C2-C10

const CARD_MAP: Record<string, React.FC<any>> = {
  candidate_list:     C1_CandidateList,
  candidate_profile:  C2_ProfileCard,
  comparison:         C3_ComparisonCard,
  job_detail:         C4_JDCard,
  job_profile:        C5_JobProfileCard,
  market_analysis:    C6_MarketAnalysisCard,
  pipeline_report:    C7_PipelineReportCard,
  interview_kit:      C8_InterviewKitCard,
  memory_recall:      C9_MemoryRecallCard,
  clarification:      C10_ClarificationCard,
};

export function CardRenderer({ card }: { card: AgentCard }) {
  const Component = CARD_MAP[card.card_type];
  if (!Component) {
    // Fallback: 未知类型渲染为 clarification 卡并附错误提示
    return <C10_ClarificationCard
      card_type="clarification"
      mode="error"
      title="未知卡片类型"
      empty_hint=""
      error_hint={`系统无法渲染卡片类型: ${card.card_type}`}
      actions={[]}
      timestamp={Date.now()}
      prompt="请重新描述您的需求"
      options={[]}
    />;
  }
  return <Component {...card} />;
}
```

**选择理由**：
- 不需要 dynamic import — 10 个卡片体积不大，tree-shaking 足够
- 静态 map 类型安全（card_type 是字面量联合类型）
- Fallback 走 C10 的 error mode，符合 ADR-007（优雅失败）
